## 🚀 面试准备检查

### 技术问题准备
- [ ] 为什么用 Beta 分布模拟 SPS?
  - **答**: Beta 分布适合模拟 [0,1] 区间的数据,通过参数调整可以控制分布形状,符合真实电商 SPS 集中在 3.8-4.5 的特点。

- [ ] 为什么用对数正态分布模拟订单量?
  - **答**: 电商订单量呈现长尾效应 (少数大卖家,多数小卖家),对数正态分布能很好地模拟这种分布。

- [ ] 95% 置信区间是怎么计算的?
  - **答**: 基于 ROAS 的标准差,使用 `mean ± 1.96 * std` 计算上下界。

- [ ] 如何对接 TikTok Shop API?
  - **答**: 使用 OAuth 2.0 认证,调用 Shop Performance API 获取 SPS 数据,每 5 分钟缓存一次。

### 业务问题准备
- [ ] SPS < 3.5 为什么是 P0 Critical?
  - **答**: 低于 3.5 会失去 Smart Promo 5x ROI 保障资格,直接影响广告效果和营收。

- [ ] 为什么要区分物流问题和质量问题?
  - **答**: 物流问题 (春节/红海) 属于 Force Majeure,可以申诉;质量问题不可申诉,需要立即下架审查。

- [ ] Smart+ 熔断器的业务价值是什么?
  - **答**: 防止 AI 广告在流量异常期间失控烧钱,每年可节省 $600,000 的亏损预算。

### 软技能准备
- [ ] 为什么承诺春节全勤?
  - **答**: 春节是跨境电商的高风险期,北京团队放假,我作为统计学专业学生,有能力也有意愿在这个关键时期守护业务。

- [ ] 如果入职,你会怎么做?
  - **答**: 第一周对接 API,第二周接入 LLM,第三周搭建数据库,8 周内将 MVP 扩展为生产级系统。
